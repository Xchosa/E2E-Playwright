# Challenge 3 — AI-Assisted Testing

**Duration:** 15 min

**App:** [Playwright TodoMVC Demo](https://demo.playwright.dev/todomvc/#/)

**File:** `tests/ai-generated.spec.ts`

**Documentation:** [test-agents](https://playwright.dev/docs/test-agents)

## Rules

- You may use AI tools / agents during this challenge
- Do **NOT** blindly trust generated code
- Every `expect()` needs an `await`
- No `waitForTimeout()`
- One behavior per test
- Review every selector and assertion generated by AI

## Before you write a single line

**Spend 5 minutes exploring the app manually.**

Try:
- adding todos
- completing todos
- deleting todos
- filtering Active / Completed

## Your goal

In this challenge, you will work with Agents in 3 roles:

1. **Planner** → generate a testing strategy
2. **Generator** → generate Playwright tests
3. **Healer** → debug and improve failing tests

---

## Step 1 — Planner

Ask your AI tool to generate a Playwright E2E testing plan for the app.

**Prompts to use**
> Generate a testing plan for this TodoMVC app.

**Your task**

---

## Step 2 — Generator

Choose **ONE** scenario from your testing plan.

**Examples:**

- adding a todo
- adding multiple todos
- deleting a todo
- completing a todo

Use AI to generate a Playwright test for that scenario.

**Example prompt**
> Generate a test that add Multiple Todos.

---

## Step 3 — Review the generated code

Before running the test, review:

- selectors
- assertions
- readability
- missing edge cases
- possible flaky behavior

**Ask yourself:**

- Does this test really verify the behavior correctly?
- Is anything missing?
- Would this test be reliable in production?

---

## Step 4 — Healer

If the test fails:

- debug the issue
- improve the prompt
- fix selectors manually
- improve assertions

**Example healer prompt**
> This test is failing. Analyze the issue and suggest a more reliable version.
